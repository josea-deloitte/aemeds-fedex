# Cards

A responsive grid of cards. Each authored row is one card with an image and a body.

## Authoring

Create a table named `Cards` with **two columns** per row:

| Cards (promo) | |
| --- | --- |
| *[image]* | ### Find many fast delivery options<br>When every minute matters, use expedited FedEx services.<br>[Explore fast shipping](/en-us/shipping/express.html) |
| *[image]* | ### Pause your deliveries<br>Place a free vacation hold on packages for up to 14 days.<br>[Pause your deliveries](/en-us/shipping/hold-at-location.html) |

| Column | Content |
| --- | --- |
| 1 | Card image. Landscape; **alt text is required**. |
| 2 | A Heading 3, a short description, and a link as the **last paragraph**. |

## Variants

- `Cards` — plain bordered cards (boilerplate look).
- `Cards (promo)` — FedEx homepage style: elevated shadow cards with rounded corners, 3:2 images, and a bold purple link pinned to the bottom of every card so links align across the row.

## Recommendations

- Three cards per row is the target layout on desktop; the grid adapts automatically below that.
- Keep descriptions similar in length so cards stay visually balanced.
- Card link text should describe the destination ("Explore fast shipping"), not "Learn more".
